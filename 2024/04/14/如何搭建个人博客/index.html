hexo和github搭建博客

本教程是教大家如何用hexo +github 搭建自己的个人博客，文章含有转载出处，教程会不断更新完善。

#安装node.js

下载 node.js

教程：https://blog.csdn.net/weixin_52799373/article/details/123840137 （过程详细）

注意一：

如果在查询版本时node-v和npm-v 是无效cmd的话，可以先配置环境变量再配置路劲。

注意二：

这三个代码输入没有反应的话可以在npm前加c即可，后面类似的安装指令也是如此。

注意三：

全局安装express 和webpack 模块之后，生成的文件夹不一定在node_modules下，在node_global文件下找到也可。

安装git并生成和配置SSH Keys

详细教程：https://blog.csdn.net/m0_46278037/article/details/118815158

注意：到了git init 指令之前就不要进行了

github 的仓库名字形式要为 你的github名字.github.io 例如我的： CallMeXD.github.io

如果仓库名形式不是这样后面你部署成功后无法访问线上。

安装hexo

1.首先需要建立博客文件夹，建议建在非系统盘，例如 ~:/Blogroot/，那么这个目录就是我们博客的根目录了。
因为每个人的命名习惯不同，本帖之后会以 [Blogroot] 指代博客根目录。

2.使用 npm 安装 Hexo, 在 [Blogroot] 路径下右键 ->Git Bash Here, 输入

npm intstall hexo-cli -g指令不起作用可在npm前面加c ，教程中的npm安装指令都可尝试加c

3.初始化 Hexo 博客：

如果hexo指令没有起作用可以使用 rpx hexo init 后面的hexo指令也是如此

然后在浏览器中打开 localhost:4000 , 就能看到

长按 Ctrl + c 关闭服务器

部署到github

在 [Blogroot] 路径下找到_config.yml 打开拉到最后面找到deploy

可以看到上面为什么要你仓库名要是 你的github名字.github.io 这种形式。

安装hexo-deployer-git 自动部署发布工具

生成页面

本地文件上传到Github上面

如果出现FATAL Error:Spwan failed 可多次尝试 hexo d

中间会出现一个登录界面，可以用令牌登录。（令牌及时保存，就看不到了）

结束以后就上传 Github 就成功了

浏览器输入 你的github名字.github.io 浏览线上页面

至此属于你的博客已经搭好，进一步美化教程在下方。

搭建和美化网站会遇到不少困难很花时间，请耐心下去。有问题可以在主页联系，或下方评论。
